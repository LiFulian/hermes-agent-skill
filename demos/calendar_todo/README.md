# 日历待办记忆增强 Demo

这个Demo展示Universal Memory Skill如何通过模拟用户创建日程待办场景，让AI智能体跨会话记住用户偏好。

## 运行Demo

```bash
# 方式1: 使用Python直接运行
cd /Users/lfl/Documents/软件开发/hermes\ agent\ skill
PYTHONPATH=. python3 demos/calendar_todo/demo_runner.py

# 方式2: 运行测试验证
PYTHONPATH=. pytest demos/calendar_todo/test_demo.py -v
```

## 场景说明

### 场景1: 首次创建会议
用户首次使用，建立基础偏好（Google日历、工作日上午、提前30分钟提醒）

**提取结果**: 7条记忆条目
- 会议信息：产品团队Q2规划会议
- 时间偏好：周一上午10点
- 提醒偏好：提前30分钟
- 日历系统：Google日历

### 场景2: 创建待办任务
AI展示已记住的偏好，并根据新交互更新记忆

**提取结果**: 6条记忆条目
- 待办与Google日历集成
- 重要事项提前1天提醒
- 下午处理文档工作

### 场景3: 设置重复提醒
发现用户的定期模式和流程

**提取结果**: 11条记忆条目
- 每周一下午3点例会
- 周五下午发周报给团队
- 周报流程：周一收集→周五发送

### 场景4: 批量创建一周日程
AI主动应用所有记忆，个性化推荐日程安排

**提取结果**: 10条记忆条目
- 根据记忆推荐时间安排
- 避免与已有日程冲突
- 应用用户的提醒偏好
